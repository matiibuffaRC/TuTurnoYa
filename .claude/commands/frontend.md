# Frontend Visual Improvement

Mejorá visualmente el componente o página indicada en $ARGUMENTS siguiendo estas reglas:

## Stack
- React + Tailwind CSS v4
- Fuente: montserrat-alternates (ya cargada globalmente)
- Sin emojis en ningún caso
- Sin comentarios en el código

## Estilo visual del proyecto
- Paleta: slate (fondos, bordes, texto), blanco para cards, colores de acento con moderación
- Bordes redondeados: rounded-xl o rounded-2xl
- Sombras sutiles: shadow-sm
- Transiciones: transition-colors o transition-all duration-200/300
- Tipografía: font-bold / font-semibold para jerarquía, text-slate-800 principal, text-slate-500 secundario
- Botones primarios: bg-slate-800 text-white hover:bg-slate-700 rounded-full
- Botones secundarios: border-2 border-slate-200 hover:border-slate-400 rounded-full
- Estados activos/seleccionados: border-slate-800 bg-slate-50
- Inputs: border-2 border-slate-200 rounded-xl focus:border-slate-800 focus:outline-none
- Cards: bg-white border border-slate-100 rounded-2xl shadow-sm

## Qué mejorar
1. Jerarquía visual clara (títulos, subtítulos, labels)
2. Espaciado consistente (gap-4, gap-6, py-3, px-5, etc.)
3. Estados vacíos con diseño limpio (sin emojis, con iconos SVG si aplica)
4. Feedback visual en hover/focus/disabled
5. Layout responsivo (grid en desktop, stack en mobile)
6. Consistencia con el resto del proyecto
